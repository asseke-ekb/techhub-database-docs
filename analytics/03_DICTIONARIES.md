# Справочники значений TechHub

## 1. Роли пользователей

### 1.1 Системные роли (account_user.role)

| Код | Название | Описание | Права |
|-----|----------|----------|-------|
| `user` | Пользователь | Обычный пользователь системы | Базовые права: просмотр, подача заявок |
| `admin` | Администратор | Администратор с расширенными правами | Управление контентом, модерация |
| `superadmin` | Суперадминистратор | Полный доступ к системе | Все права, включая системные |
| `moderator` | Модератор | Модерация контента | Одобрение/отклонение событий, вакансий, блогов |
| `expert` | Эксперт | Эксперт для оценки заявок | Экспертиза заявок на услуги |
| `manager` | Менеджер | Менеджер услуг | Управление заявками на услуги |
| `cett_expert` | Эксперт ЦЭТТ | Эксперт центра экспертизы | Экспертиза в рамках ЦЭТТ |
| `cett_moderator` | Модератор ЦЭТТ | Модератор ЦЭТТ | Модерация заявок ЦЭТТ |
| `techorda_manager` | Менеджер TechOrda | Управление образовательными программами | Управление TechOrda |
| `niokr_admin` | Админ НИОКР | Администратор научных проектов | Управление НИОКР |

### 1.2 Роли в компании (account_usercompany.role)

| Код | Название | Описание | Права |
|-----|----------|----------|-------|
| `owner` | Владелец | Владелец компании | Полные права. **Только один на компанию** |
| `member` | Сотрудник | Сотрудник компании | Действия от имени компании |
| `advisor` | Консультант | Консультант/советник | Ограниченный просмотр |

### 2.4 Структура тега компании

```json
{
  "status": "active",        // Статус тега: active, inactive, pending, rejected
  "date_from": "2024-01-15", // Дата получения статуса
  "date_to": null,           // Дата окончания (null = бессрочно)
  "data": {                  // Дополнительные данные
    "certificate_number": "123",
    "approved_by": "user_id"
  }
}
```

### 2.5 Статусы тегов

| Код | Название | Описание |
|-----|----------|----------|
| `active` | Активный | Статус действует |
| `inactive` | Неактивный | Статус приостановлен |
| `pending` | На рассмотрении | Заявка на рассмотрении |
| `rejected` | Отклонён | Заявка отклонена |

## 3. Статусы контента

### 3.1 Статусы публикации (event, vacancy, blog, article)

| Код | Название | Описание | Следующие статусы |
|-----|----------|----------|-------------------|
| `draft` | Черновик | Создан, не отправлен на модерацию | sent |
| `sent` | На модерации | Отправлен на проверку модератором | success, correction, reject |
| `correction` | На доработке | Требуется исправление автором | sent |
| `success` | Опубликован | Прошёл модерацию, опубликован | deleted |
| `reject` | Отклонён | Отклонён модератором | — |
| `deleted` | Удалён | Удалён автором или модератором | — |

### 3.2 Диаграмма переходов

```
                                 отправить
        ┌─────────┐ ─────────────────────────────→ ┌─────────┐
        │  draft  │                                │  sent   │
        │(черновик)│ ←───────────────────────────── │(модерац.)│
        └─────────┘      вернуть в черновик        └────┬────┘
                                                        │
                         ┌──────────────────────────────┼──────────────────────────────┐
                         │                              │                              │
                         ▼                              ▼                              ▼
                  ┌───────────┐                  ┌──────────┐                   ┌──────────┐
                  │correction │                  │ success  │                   │  reject  │
                  │(доработка)│                  │(опублик.)│                   │(отклонён)│
                  └─────┬─────┘                  └──────────┘                   └──────────┘
                        │
                        │ исправить и отправить
                        ▼
                  ┌─────────┐
                  │  sent   │
                  └─────────┘
```

## 4. Статусы заявок на услуги

### 4.1 Статусы заявок (service_servicerequest.status)

| Код | Название | Описание | Следующие статусы |
|-----|----------|----------|-------------------|
| `draft` | Черновик | Заявка создана, не отправлена | sent |
| `created` | Создана | Заявка создана (альт. начальный статус) | sent |
| `sent` | На рассмотрении | Отправлена на проверку | approved, correction, reject |
| `correction` | На доработке | Требуется исправление | sent |
| `approved` | Одобрена | Предварительно одобрена | success, reject |
| `success` | Успешно завершена | Заявка полностью одобрена | — |
| `reject` | Отклонена | Заявка отклонена | — |
| `rejected` | Отклонена (альт.) | Альтернативный код отклонения | — |

### 4.2 Жизненный цикл заявки

```
┌─────────┐     отправить     ┌─────────┐     проверка     ┌──────────┐     экспертиза    ┌──────────┐
│  draft  │──────────────────→│  sent   │─────────────────→│ approved │───────────────────→│ success  │
│(черновик)│                   │(рассмотр)│                  │(одобрено)│                    │(успешно) │
└─────────┘                   └────┬────┘                   └────┬─────┘                    └──────────┘
                                   │                              │
                                   │ доработка                    │ отклонение
                                   ▼                              ▼
                            ┌───────────┐                  ┌──────────┐
                            │correction │                  │  reject  │
                            │(доработка)│                  │(отклонено)│
                            └─────┬─────┘                  └──────────┘
                                  │
                                  │ исправить
                                  ▼
                            ┌─────────┐
                            │  sent   │
                            └─────────┘
```
